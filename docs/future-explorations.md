# Future Explorations

This document contains a list of tools and technologies that you might find interesting to explore in the future to further enhance your development workflow.

## Git

*   **[lazygit](https://github.com/jesseduffield/lazygit):** A simple terminal UI for git commands. It allows you to easily add files, commit, and manage your git history in a more visual way. It integrates very well with Neovim.

## API Testing

*   **[httpie](https://httpie.io/):** A command-line HTTP client that is very user-friendly and provides a more readable output than `curl`.
*   **[curlie](https://github.com/rs/curlie):** A frontend to `curl` that adds the ease of use of `httpie` with the power and features of `curl`.

## Web Development

*   **[bun](https://bun.sh/):** A fast, all-in-one JavaScript runtime, bundler, transpiler, and package manager. It can be a faster alternative to Node.js and npm/pnpm for many use cases.
*   **[esbuild](https://esbuild.github.io/):** An extremely fast JavaScript bundler and minifier.
*   **[swc](https://swc.rs/):** A super-fast TypeScript/JavaScript compiler written in Rust.

## Python

*   **[poetry](https://python-poetry.org/):** A tool for dependency management and packaging in Python. It helps you declare the libraries your project depends on and it will manage (install/update) them for you.
*   **[rye](https://rye-up.com/):** An all-in-one tool for Python development that can manage Python installations, virtualenvs, dependencies, and more.

## Code Formatting and Linting

*   **[prettier](https://prettier.io/):** An opinionated code formatter that supports many languages.
*   **[eslint](https://eslint.org/):** A static analysis tool for identifying and reporting on patterns found in ECMAScript/JavaScript code.

## Database Clients

*   **[psql](https://www.postgresql.org/docs/current/app-psql.html):** The official command-line client for PostgreSQL.
*   **[mysql-client](https://dev.mysql.com/doc/refman/8.0/en/mysql.html):** The official command-line client for MySQL.
*   **[redis-cli](https://redis.io/topics/redis-cli):** The command-line interface for Redis.
